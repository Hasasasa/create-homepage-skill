# 自测场景（回归 + 触发）

改完这个 skill 后拿这 5 个场景对照自检：想象用户这么说，走一遍流程，看 skill 会不会做出「期望行为」。个人站是主观产出、不做自动打分，这几条是**防退化**和**验证触发**的定性防线。

---

## 场景 A · 用户带着 brief + 参考来

**用户**："帮我做个个人作品集，我是做产品的，想要暗色、高级、有科技感但不要像模板。照着 xxx.com 首页那个滚动的感觉做。React 就行。"

**期望行为：**
- 阶段 0 **接住并复述** brief（身份=产品、气质=暗色高级科技感、参考=xxx.com、栈=React、硬约束里确认脱敏），缺的关键项一次问一个——**不**假装用户是白纸从零盘问。
- 若环境有 skill 机制，探测可复用的设计/动效/文案 skill（没有就跳过）。
- 阶段 1 **先出能跑的灵动基础版**（铺连接层），再给用户看。
- 阶段 2 走"先找后造"：**以用户的 xxx.com 参考为准**，按三点拆解，不拿默认款覆盖。

**红线（出现即退化）：** 无视用户已给的 brief 和参考，从零访谈发明隐喻；一上来甩一堆问题；先给线框不给能跑的灵动基础版。

---

## 场景 B · 小白，啥都没带、没有隐喻想法

**用户**："我想做个个人主页，我是做运营的，但我不知道要搞成什么样，你帮我弄一个好看点的。"

**期望行为：**
- 触发本 skill（用户没说"动效/作品集/React"也应触发）。
- 阶段 0 探测本地 skill，只锁硬约束。
- 阶段 1 **先用连接层出一个已经灵动、协调、能跑的基础版**——即使用户没带任何参考，底子也不空洞不单板（这是连接层存在的意义）。
- 阶段 2 走"先找后造"：**先引导他去 Awwwards/reactbits/paper 带回 2–3 个"想要这种感觉"的例子**；实在带不回，才**判断要不要隐喻**：撑得起就从"运营"世界共创 2–3 个候选让他挑，撑不起就走直白的编辑型方向（证据结构 + 排版性格 + 一个签名交互）——不硬凑世界观，但也不做平庸默认款。
- 阶段 3 主动要真实源文档抠证据，别让内容停在通用套话。

**红线：** 逼一个白纸小白凭空发明世界观隐喻；不出基础版就干问；做出空洞单板的站。

---

## 场景 C · 要极简、静态、不要花哨动画

**用户**："别搞那些花里胡哨的开屏和粒子，我要干净、快、能上线就行。"

**期望行为：**
- **优雅降级**：背景层选静底或无，动效收敛到克制（连接层的原语可以只用 `Reveal` 的最小量，或全静）。
- 新建项目默认选**静态/轻交互路径**，不为通用淡入强上 React、framer-motion、three.js 或 paper-shaders；若用户已有站则保留原栈。
- **三层减动效契约照旧**——极简不等于省掉无障碍降级。
- **上线就绪照旧**做（SEO/OG/favicon/JSON-LD）；文案关、通病黑名单照常。

**红线：** 仍硬塞开屏动画/粒子；为一个淡入效果引入完整 React 动效栈；或因为"极简"把 SEO/分享卡 / 减动效降级省了。

---

## 场景 D · 改造已有个人站

**用户**："这是我现有的 Astro 个人站，内容不用重写，帮我把首页做得更有记忆点。"

**期望行为：**
- 先读现有结构、组件和样式约定，默认保留 Astro 与原构建流程。
- 在原栈内加入 token、签名交互和减动效契约；只有用户明确授权才迁移框架。
- 修改范围保持可回滚，不为复用 React 版 `motion-kit` 重搭项目。

**红线：** 未经授权改成 React/Vite；复制一套新脚手架覆盖现有架构。

---

## 场景 E · 同职业、防职业模板

**输入**：连续为两位同职业的人做主页，但两人的经历、作品证据、受众任务和审美不同。

**期望行为：**
- 职业名不算隐喻证据；每个方向都从个人经历、作品材料和访问者任务中寻找至少两个独立信号。
- 至少给一个不靠职业隐喻的编辑型方向。
- 两个站至少在信息主次/版块顺序、Hero 构图、访问者任务/签名交互、贯穿动线/视觉语法中的两项不同。
- 做换人、去皮、同职业反事实三层自检。

**红线：** 两站都使用“大标题 + 右侧职业演示器 + 固定顶栏 + 微型英文标签 + 网格线”的同一骨架；只换职业道具、颜色和字体。

## 触发自测

应触发（即使没点名 React/动效/作品集）：
- "帮我做个人主页 / 作品集 / 个人站"
- "把我这个简历做成一个网页放出去"
- "我想要个像 awwwards 上那种的个人网站"
- "重做一下我现有的个人站，但保留现在的技术栈"

不应触发（近义误伤）：
- "帮我给公司做个落地页/营销页"（landing page，不是个人站叙事，除非用户明确要个人作品集气质）
- "改一下我现有网页的某个 bug"（是维护，不是从头做站）
- "写一篇个人简介文案"（只是文案，没有建站意图）
