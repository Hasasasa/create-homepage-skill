# 素材与账号清单（向用户索取）

不要在基础版前把整份清单一次发给用户。阶段 0 只收最小内容集：身份、一句话定位、2–3 条代表性证据或经确认的设定，以及隐私级别；下面其余内容在对应阶段按需补，一次问一个。内容与视觉决策要有单一事实来源，并沿用所选技术路径或现有项目的组织方式。

## 内容素材

- **身份**：中文名、英文名 / handle、一句话定位（职位）、经验年限（如需拆分：X 年项目管理 + Y 年产品，问清口径——"总工龄"和"某岗位年限"是两回事）。
- **关于我**：3 段左右的介绍，或给要点让你起草（起草后必须回给用户确认语气）。
- **经历 / 项目时间线**：每条给 `时间 / 标题 / 角色 / 一句话描述 / 链接`。正职与副业/开源**分开给**（对应主线/支线）。**优先给能证明价值的少数几条**，每条带**可量化成果**（数字、上线、增长），角色如实不夸大——proof over polish，见 `interview.md` Q9。
- **实时数据源**：只收与访问者任务有关的公开指标和具体来源，例如用户指定仓库的 star；记录更新时间与失败展示方式，不默认拉取或汇总全部账号数据。
- **社交与联系**：平台 + 链接（B站/YouTube/GitHub/App Store/小红书……）、联系邮箱。
- **图片 / logo**（可选）：文件路径。**提醒**：会进入站点交付物或代码仓库；若仓库/站点将公开，先确认肖像权、版权与隐私范围。用不到就别塞进 public（每次构建都打包）。
  - **先分诊再锁布局**：拿到用户的图，逐个按角色归类（hero 人像 / 作品截图 / logo / 二维码 / 纹理底 / 产品视频），并标**能不能用 + 原因**——例如"小尺寸截图铺满宽 hero 会糊，不可用""竖图当横 banner 不合比例"。不可用的：让用户换图、改布局迁就、或用生成式封面/纯色兜底。别拿到图就硬塞进设计里。
  - **公开前查元数据**：检查原图、缩略图和分享卡中的 EXIF/GPS、设备、作者、软件、注释等嵌入字段；按隐私清单移除敏感项，同时保留许可证或法律要求的版权/署名信息。不要只改文件名就当作已脱敏。

## 证据或设定来源（优先公开、已脱敏或经确认）

真实人物优先使用公开页面、已脱敏摘录或用户自己的事实总结。不要默认索要原始合同、聊天记录、会议纪要；先确认用户有权向当前 AI/工作区披露，受 NDA、客户保密或第三方隐私约束的材料不要上传。虚构人物使用用户确认的人设与作品清单，不要求伪造真实来源。

- **公开项目页 / 已脱敏复盘摘录** → 抽取可量化成果。
- **用户总结的角色边界** → 还原具体做了什么、不夸大团队成果。
- **确有授权的源文档** → 仅提取必要片段；不写入搜索请求、日志或公开仓库。
- **经用户确认的虚构人设 / 作品清单** → 作为单一事实来源；新增履历事实前再次确认，不伪造“真实证据”。
- **live 数据源** → 优先官方 API；第三方代理需另查可靠性、条款与隐私影响。只有来源稳定、失败状态透明且值得维护时才接，缓存值标注更新时间，不用无来源数字假装实时。

处理后，把职责换成结果、区分团队成果和个人贡献。临时副本只在任务需要期间保留，完成后清理本轮创建的副本；不要删除用户原有文件。

## 规则类（必须写下来）

- **隐私分级清单**：逐项标记可公开原名 / 需泛化 / 禁止出现，并记录批准的替代称呼。
- **文案语气**：正式书面 / 口语化；语言（中文 / 双语）。定稿前所有正文过一遍去 AI 腔检查（装了 `avoid-ai-writing` 就用它，否则照其原则自查）。
- **视觉偏好**：主色/强调色、从参考站带回的 2–3 个例子；想模仿的动效参考（站链接 / 效果截图 / 现成组件代码），逐个标"想要它的哪一点"。

## 部署相关（阶段 5 才用，可先问清）

- **交付选择**：仅本地 / 现有私有仓库 / 新仓库（私有或公开）/ 部署；没有明确选择就不执行外部操作。
- **现有托管**：已有平台、构建设置、域名和 DNS；改站优先沿用。面向中国大陆时按实测、备案条件和合规要求选服务，不预设普通境外 Pages 可达性。
- **账号授权**：只有选定对应平台后才进行；登录、创建仓库、设为公开、部署和 DNS 修改分别确认。

## 事实类（自己查，别问用户）

- 已上架 App 的上线时间、评分 → App Store lookup API（`itunes.apple.com/lookup?id=...`）。
- 仓库的 star / 创建时间 / 描述 → 优先 GitHub 官方 API；只有确认条款、可靠性和隐私边界后才用第三方代理。
- 现有代码里的结构、样式、历史决策 → 直接读文件。
