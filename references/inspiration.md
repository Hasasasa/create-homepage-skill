# 参考与动效来源

两件事：**隐喻/审美从哪找**（先找后造，且隐喻可选），**具体动效从哪取**（仅在所选技术路径需要时现取，并过连接层）。

无论哪条路，参考站**只用来校准气质与手感、或作为可移植的效果来源，不是拿来抄版式**——最终视觉必须从这个人的专属世界观里长出来。

---

## 先探测复用本地 skill（先看，少造轮子）

动手前，**若你所在环境有 skill 机制，先看有没有能直接用的 skill**——很多美化/设计的活别人已经封装好了。环境没有 skill 机制（或你是别的 agent 框架）就跳过本节。

**怎么探测：** 以 Claude Code 为例，列出 `~/.claude/skills/` 和项目级 `.claude/skills/` 下的目录，对每个读它的 `SKILL.md` frontmatter 描述（或看名字），挑出这几类：
- **设计系统 / UI 风格**（premium、expressive、professional、clean、bento、shadcn 等）
- **动效 / 组件**
- **去 AI 腔文案**（如 avoid-ai-writing）
- **Figma / 视觉生成**

**怎么复用：**
- 设计系统 skill → 给阶段 0 校准提供词汇和起点（"要 premium 的克制，还是 expressive 的张扬？"），把它的**配色 / 字阶 / 间距 / 圆角约定**当 `theme.js` 的起点，再叠加这个人的强调色。
- 文案 skill（如 avoid-ai-writing）→ 装了就在正文定稿前必过；没有就按去 AI 腔原则自查（铁律 5）。
- 动效 / 组件 skill → 作为实现技法参考，落地仍要过连接层；不要把现成模块外壳当作个人站结构。
- 探测到就**告诉用户"你已经装了 X，我打算用它来做 Y"**让他确认。

## 隐喻 / 审美：先找后造

### 先找 —— 优先用户自带参考
真实项目里那个"航行/纸飞机"世界观，就是用户自己贴了 `paperplanes.world` 才出现的，不是访谈现场发明的。所以**第一步永远是问用户有没有想模仿的站/截图/组件代码**。有就以它为准。

### 引导找 —— 没有就去这些站带回 2–3 个例子
- **awwwards.com** — 获奖网站画廊，判断"什么叫做得好"的基准线。看 Sites of the Day 和 Portfolio 分类。挖：整体大胆程度、滚动叙事结构、排版的主角地位、留白节奏。
- **siteinspire.com** — 看**版式结构**。
- **pinterest / behance** — 看**气质与品牌调性**。
- 让用户挑 2–3 个向往的，逐个问"你喜欢它哪一点"——配色？字体？某个滚动效果？还是整体气质？

### 才共创 —— 实在没有才推导，且先判断需不需要隐喻
**隐喻是可选的，不是每个人都要有。** 先看这个人**撑不撑得起隐喻**。职业名称本身不计为支持信号；至少要有两个彼此独立、来自职业之外的信号，例如：个人经历中的转折、作品反复出现的材料/动作、真实文档的结构、访问者要完成的任务、本人惯用的语言。两个同义关键词不算两个信号。

- **撑得起** → 从上述个人证据提炼 2–3 个候选，同时给一个不靠隐喻的编辑型候选。模型最先想到的职业联想默认是高风险项；除非个人证据能把它变得具体，否则舍弃。
- **撑不起**（以文字/证据取胜的顾问、研究者，或硬套就会刻板化/让证据更难扫读的人）→ **走直白的编辑型方向**：不套主题外壳，独特性来自「用最强证据定内容主次 + 有性格的字体排版与留白构图 + 一个签名交互」。一样能做出只属于他的站。

**该拒绝隐喻的信号**：纯为了显得聪明、把职业刻板化、让项目证据更难扫读、需要用户根本不会说的装饰性文案。两条路最后都要过铁律 10 的**换人 / 去皮 / 同职业反事实**三层自检——无隐喻 ≠ 平庸。

### 拆解每个参考（自带的或选中的）
拿到一个参考，**别整体照搬**——逼自己拆成三点写下来：
1. **喜欢它哪一点**：具体到配色 / 某个滚动效果 / hover 手感 / 排版气质，不是"整体好看"。
2. **绝不照抄哪一点**：明确一条不碰的，避免站变成它的翻版（也顺带避开它可能带的 AI 通病版式）。
3. **可迁移到这个人身上的想法**：把那"喜欢的一点"翻译成他世界观里的东西。

**收尾自检（铁律 10）**：
1. **换人**：换掉名字与主色还能套给别人吗？
2. **去皮**：去掉配色、字体与职业文案，是否仍是同一套 Hero、导航、信息顺序和交互骨架？
3. **同职业反事实**：换成另一个同职业、但经历/受众不同的人，是否仍可原样套用？

多个候选至少要在**信息主次与版块顺序 / Hero 构图 / 访问者任务与签名交互 / 贯穿动线与视觉语法**中的两项不同。大标题左、概念演示器右、固定顶栏、微型英文标签、细网格线等元素并非禁用，但不能成套作为每个人的默认骨架。

## 具体动效：按技术路径现取

静态/轻交互路径先用 CSS 或少量原生 JS；不要为了库组件反向引入 React。只有选中 React 强交互路径且确实需要某个效果时，才从下面两个库**按需取**，看懂原理后移植，并**过连接层**（`motion-kit.md`：统一 token + 补三层减动效/性能契约）。**不预置固定动效引擎**。

> 只用这两个来源是有意的：reactbits 许可证清晰（MIT + Commons Clause，个人站可免费用，仅需保留版权注释）、paper 是正规 npm 包。**避开逐个组件许可证不明的组件市场**——公开求职站用来路不清的代码是隐患。

### reactbits.dev —— 首选，最贴栈
动画化 React 组件集，**拷贝源码即用**，天生贴 framer-motion / Canvas。
- Text animations：SplitText（逐字入场）、Scramble、Glitch、Decrypted、Shiny、Gradient——标题动效现成方案。
- Backgrounds：各种粒子/网格背景。
- 卡片辉光、磁吸、3D 倾斜等微交互。
- **移植注意**：几乎零改造，但要补连接层第 5 节的减动效三层。**拷了实质源码就保留其版权注释**，并在用户项目里的 `THIRD_PARTY_NOTICES.md` 记一句来源/作者/许可证。

### shaders.paper.design —— 生成式 WebGL 背景/封面
`npm i @paper-design/shaders-react`（`MeshGradient` / `Warp` / `GrainGradient`）。mesh / warp / grain 等 shader 作动态封面或氛围背景，不用手写着色器就拿到高级生成式视觉。
- **移植注意**：WebGL 较重，**懒加载、控制弱机开销、切后台暂停**。作品无真实截图时的动态封面、或全局氛围底很好用。

### 手感对标
reactbits 定义"灵动"的手感——弹簧物理、有回弹、有余温，不是线性缓动的死板过渡。照它的手感调 `theme.js` 的 spring 预设（stiffness / damping / mass）。

**来源速查**（不同站看不同维度）：reactbits 看**可直接移植的效果与微交互手感**、paper 看**生成式背景**、Awwwards 看 **hero 冲击力与张扬度**、SiteInspire 看**版式结构**、Pinterest/Behance 看**气质调性**。
